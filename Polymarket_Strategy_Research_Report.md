### 1.2 browomo - "垃圾收集者"

#### 核心策略
**反向思维套利 - 押注"不可能事件"**

> "这个钱包通过押注永远不会发生的事情，把 $5 变成了 $370 万"

#### 策略分解

**步骤 1: 寻找错误定价市场**
```
- 寻找赔率 > 90% "不会发生" 的选项
- 这些选项被市场忽视，流动性低
- 价格可能远低于真实概率
```

**步骤 2: 数学验证**
```
- 使用 Claude Code 编写分析脚本
- 计算真实概率 vs 市场定价概率
- 寻找 Expected Value (EV) > 0 的机会
- 脚本运行 12 分钟找到目标
```

**步骤 3: 小资金测试 + 规模化**
```
- 初始投入: $5
- 验证策略有效性
- 逐步增加仓位
- 最终收益: $3.7M
```

#### 关键数据
| 指标 | 数值 |
|------|------|
| 初始投入 | $5 |
| 最终收益 | $3.7M |
| 回报率 | 74,000,000% |
| 技术工具 | Claude Code |
| 策略类型 | 反向套利 |

#### 心理学基础
1. **市场偏见**: 人们喜欢押注"会发生的"
2. **反向共识**: 押注"不会发生"获得极高赔率
3. **长尾收益**: 小概率 × 高赔率 = 正期望值

### 4.2 核心模块设计

#### 做市策略核心算法

```python
class MarketMakingEngine:
    def calculate_fair_price(self, orderbook):
        """
        计算公平价格
        - 基于订单簿加权
        - 考虑交易历史
        - 事件影响调整
        """
        mid_price = (best_bid + best_ask) / 2
        trend_adjustment = self.calculate_trend()
        inventory_skew = self.calculate_inventory_skew()
        return mid_price + trend_adjustment + inventory_skew
    
    def dynamic_spread(self, volatility, inventory):
        """
        动态价差调整
        - 波动率高 → 价差扩大
        - 库存偏向 → 价差倾斜
        """
        base_spread = volatility * k1
        inventory_adjustment = abs(inventory) * k2
        return base_spread + inventory_adjustment
```

## 第六部分：风险与合规

### 6.1 风险提示

1. **市场风险**: 预测市场波动大，可能亏损本金
2. **技术风险**: 自动化系统可能出错
3. **流动性风险**: 小市场可能无法及时成交
4. **监管风险**: 不同地区法规不同

### 6.2 风险管理建议

- 只投入可承受损失的资金
- 设置严格的止损机制
- 分散多个市场和策略
